The keystone

Control the loop — a gateway can't.

Most "agent gateways" sit in front of a black-box loop: they see the request and the final answer, nothing in between. airlock is a runtime. Its Loop Engine makes the model calls and dispatches the tools itself, emitting a StepEvent per iteration and consuming ControlSignals between steps. That's the only place step-level control exists — and it's why a front-of-agent gateway structurally can't copy it.

  • model → tool_result → final, one ordered step at a time
  • Every step carries tokens, cost, duration, status — recorded live
  • Between any two steps: continue · pause · retry · kill · override

Control surface

Production controls, applied per step.

Declared once in worker.yaml, enforced by the engine on every iteration of the loop.

⏿

Budget & step caps

Hard token / step / dollar ceilings. A run that blows the budget stops mid-loop with a recorded stop_reason — not after the bill.

⛋

Tool gating & approval

Gate a tool by its arguments, or hold a side-effecting call for a human to approve, deny, edit, or override — then resume the exact run.

⇄

Model routing & fallback

A per-run default, whole-run variant switching, and automatic retry-then-backup when a model binding fails. Heavy step to a big model, cheap step to a small one.

▤

Sandboxed execution

Every tool runs in a subprocess with CPU / memory / wall-clock limits and no host network unless you grant it. A runaway tool is killed, not trusted.

◷

State, cache & resume

Pluggable state store (SQLite by default) keyed tenant-first. Tool-result cache, session continuity, and resume / fork of a paused run.

⛨

I/O contract

Input guards (prompt-injection denylist) reject before any model call; output redaction scrubs the answer. Typed /skills/<id> surface with on/off ACLs.

Harness-agnostic

Bring the framework you already use.

airlock extracts a framework's tools, planner, and prompt assembly and runs the loop itself — so the full control set applies no matter who wrote the agent. How much control a harness gets is feature-derived: it depends on how much of the loop airlock can own.

HarnessWhat it isControl mode
langgraphLangGraph agentsOWN · full control
smolagentsHF smolagentsOWN · full control
crewaiCrewAI crewsOWN · full control
openai-agentsOpenAI Agents SDKOWN · full control
claudeClaude Agent SDKOWN · full control
openaiairlock's own loop over any OpenAI-compatible endpointOWN · full control
stubdeterministic, scriptable test bindingOWN · full control
customyour own callable, or a PlannerTerminal / WRAP · observe-only*

*A plain custom callable is opaque, so airlock sees only the final result. Implement the Planner protocol or expose extractable tools and it becomes OWN — the full control set applies.

Internal = external

Flip the same worker to the public web.

A worker runs as a stable internal service other agents and apps call. Set expose: public and airlock opens a Cloudflare Tunnel — the same worker, same routes, same controls, now on a real public https:// URL. No rebuild. Internal vs external differs only in network binding + auth.

  • Internal: a stable service in your private network
  • Public: a real *.trycloudflare.com URL (zero account) or your own domain (durable tunnel)
  • The model stays yours — airlock never hosts inference
# the same worker.yaml — one line decides reach
harness: openai
expose: public        # ← internal | public

controls:
  budget: { tokens: 40000 }
  approvals: [{ tool: send }]

# then:
$ airlock up
✓ live at  https://<rand>.trycloudflare.com

Quickstart

From clone to a public agent.

1
Describe the worker. One worker.yaml: harness, model bindings, tools/skills, controls, and expose.
2
Run it. airlock up boots the worker on your hardware and opens a real Cloudflare quick tunnel — no account needed.
3
Watch it. Open /console to drive a run step-by-step, inspect traces, and clear approval holds live.
4
Prove it. bash scripts/live-proof.sh drives the whole control surface over the public URL and saves a dated transcript.
# install the runtime + CLI
$ cd python/agent-runtime && pip install -e ".[dev]"
$ cd packages/cli && npm install && npm run build

# boot a worker + go public
$ cd examples/live-demo && airlock up
✓ live at  https://<rand>.trycloudflare.com
  console:  http://localhost:3000/console

# prove the whole surface on the public web
$ bash scripts/live-proof.sh
